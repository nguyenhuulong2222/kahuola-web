# V4.8 Trust Model
## Civic Trust, Data Honesty, and User Confidence Framework

**Status:** Authoritative V4.8 Trust Model

## Core Trust Principle

Kahu Ola does not ask users to trust hidden logic.

Kahu Ola builds trust by showing:
- what the hazard is
- where it matters
- when the data was checked
- where the data came from
- where to go for official action

## Trust Layers

### Layer 1 — Plain-Language Hazard Meaning
The first thing users see must be civic meaning, not technical labels.

### Layer 2 — Freshness Honesty
Every signal must show updated time, last checked time, and stale indicator if needed.

### Layer 3 — Source Attribution
Examples:
- Source: NASA FIRMS
- Source: NWS Honolulu
- Source: EPA AirNow

### Layer 4 — Confidence and Signal Type
Show confidence, satellite-only note, and watch/warning distinction where relevant.

### Layer 5 — Official Authority Boundary
Kahu Ola provides situational awareness and links to official sources.
It does not issue evacuation orders or safe-route instructions.

### Layer 6 — Degraded Transparency
If data is stale or delayed, users must be told calmly and clearly.

## Trust Failure Modes

Trust is damaged when the system:
- hides stale data
- overstates certainty
- sounds official when it is not
- shows conflicting hero states
- goes blank during outages
- mislabels one hazard as another

## Final Requirement

All Kahu Ola implementations must preserve this trust model across web, mobile, map, and notification surfaces before release.
