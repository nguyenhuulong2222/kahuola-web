Data Integrity

Operational safeguards and official follow-up

Kahu Ola uses multi-source validation across public-domain feeds, near-real-time ingestion where supported, and on-device edge processing for location-sensitive functions. The platform is a supplemental civic tool and is not a replacement for official emergency directives from Maui County or other authorized agencies.

Official sources residents should follow:

  • weather.gov (NWS)
  • ready.gov
  • tsunami.gov
  • usgs.gov (hazards)
  • airnow.gov
  • fema.gov
  • mauicounty.gov (local guidance)

Category 1

Federal & Academic Natural Hazard Sensors (Open Data)

Public U.S. hazard and environmental feeds used for situational awareness and validation.

NASA FIRMS - Public Data

NASA FIRMS (Public Open Data)

Source: firms.modaps.eosdis.nasa.gov

Data: Satellite-derived thermal anomaly detection for early wildfire situational awareness.

Supports: Fire Watch, Alert Buddy, Verified Updates.

Community Impact: Enables earlier awareness before ground-level reports reach Maui residents.

NOAA Forecast - Public Data

NOAA Weather.gov Forecast API (Public Open Data)

Source: api.weather.gov

Data: Forecast and gridpoint weather context for wind, humidity, and changing local conditions.

Supports: Weather Radar, Evac Route, Supplies Planner.

Community Impact: Helps Maui households judge fast-moving weather risk before travel or evacuation.

NOAA Alerts - Public Data

NOAA Weather.gov Alerts API (Public Open Data)

Source: api.weather.gov/openapi.json

Data: Public alert bulletin ingestion for watches, warnings, and active hazard statements.

Supports: Alert Buddy, Verified Updates, Family Check-in.

Community Impact: Keeps public-facing summaries aligned with official alert timing.

NOAA Tides - Public Data

NOAA Tides & Currents API (Public Open Data)

Source: api.tidesandcurrents.noaa.gov

Data: Coastal water level and tide reference for shoreline and harbor conditions.

Supports: Water Safety, Road Conditions.

Community Impact: Supports safer planning for coastal communities, harbor users, and low-lying areas.

USGS Seismic - Public Data

USGS Earthquake Hazards API (Public Open Data)

Source: earthquake.usgs.gov

Data: Seismic event reference and aftershock context for regional hazard awareness.

Supports: Alert Buddy, Verified Updates, Recovery Resources.

Community Impact: Adds another trusted signal when Maui residents need rapid event context.

USGS Water - Public Data

USGS Water Services API (Public Open Data)

Source: waterservices.usgs.gov

Data: Streamflow and water-monitoring references for water-related disruption scenarios.

Supports: Water Safety, Community Signals.

Community Impact: Helps residents understand changing water conditions during severe events.

FEMA Recovery - Public Data

FEMA OpenFEMA API (Public Open Data)

Source: fema.gov/about/openfema/api

Data: Disaster declarations and recovery-reference data for post-event workflows.

Supports: Recovery Resources, Shelter Finder.

Community Impact: Gives Maui residents clearer access to recovery context after major incidents.

EPA Air - Public Data

EPA AirNow API (Public Open Data)

Source: docs.airnowapi.org

Data: Air quality and smoke exposure reference data for health-aware decision support.

Supports: Air Quality, Alert Buddy.

Community Impact: Helps Maui residents make safer choices about masks, sheltering, and outdoor activity.

About Kahu Ola

Independent civic technology for Maui County

Kahu Ola is an independent civic-tech effort designed to complement, not replace, official alerts. It focuses on legibility, steady guidance, and a more careful public-facing experience during disruption.

  • References public data sources including NASA, NOAA, and USGS.
  • Support and compliance contact: long@kahuola.org.
  • Follow official Maui emergency guidance.

Project stance

Calm, authoritative, and public-serving

The portal is intentionally restrained. It avoids hype, keeps copy short, and uses accessible structure so households can find critical context faster.

Kahu Ola is built around on-device edge processing principles for location-sensitive safety features, minimizing exposure of raw precise location data by default.

Compliance

Privacy and support pages remain prominent and review-ready

Location-sensitive functions are designed so processing happens on the user side through edge methods by default. The root-domain privacy and support pages are linked below and remain available for review.
